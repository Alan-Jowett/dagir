Return a compact unique node id for a stable key.

This helper assigns sequential identifiers (node000, node001, ...) for keys seen during program execution. It is thread-safe and intended for use by policy implementations that need renderer-visible unique ids.

Parses a logical expression string into an expression tree.

Clean recursive descent parser that handles operator precedence correctly. Supports proper operator precedence (highest to lowest):

  1. NOT (unary, right-associative)
  2. AND (binary, left-associative)
  3. OR (binary, left-associative)
  4. XOR (binary, left-associative)
Parameters
expr_strString representation of the logical expression
Returns
Smart pointer to the root of the parsed expression tree

◆ read_expression_from_file()

my_expression_ptr dagir::utility::read_expression_from_file ( const std::string &  filename)
inline

Reads and parses a logical expression from a text file.

Reads all non-empty, non-comment lines from the specified file and concatenates them into a single expression string. Lines starting with '#' are treated as comments and ignored.

Parameters
filenamePath to the file containing the expression
Returns
Smart pointer to the parsed expression tree
Exceptions
std::runtime_errorIf file cannot be opened or no valid expression is found

File format:

  • One or more lines containing parts of the expression
  • Empty lines are ignored
  • Lines starting with '#' are treated as comments
  • Expression parts are concatenated with spaces

Example file content:

# This is a comment
(x0 AND x1) OR
(NOT x2) XOR (x3 AND (NOT x4))
